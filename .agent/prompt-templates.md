# Output Templates

## 1. 修正提案時
常に以下の構成で回答せよ：
1. **現状分析:** 修正対象のコードの課題。
2. **設計の意図:** 修正後のコードが「なぜ」優れているか（GEMINI.md準拠）。
3. **実装コード:** 差分または全量。
4. **検証項目:** 動作確認すべきポイント。

## 2. 実装計画作成時
`implementation_plan.md` は以下のセクションを持つこと：
- 目的 / ゴール
- 影響範囲（Frontend / Backend / Data Schema）
- ステップごとのタスクリスト
- リスクと対策
